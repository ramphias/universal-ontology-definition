# 产品需求文档 (Product Requirements Document)
**项目名称:** Ontology Studio
**项目代号:** UOD-Studio
**版本:** v1.1 (Revision: 可视化全图景编排方案)
**主题风格:** 企业极简风 (德勤黑绿色调 / 暗夜模式)

---

## 1. 产品愿景与价值定位 (Product Vision)

**核心痛点**：目前团队以 L0-L3 为核心的本体数据资产，依靠原生 JSON 和 Git 代码库手动维护，结构庞大、依赖关系复杂，导致非技术背景的业务人员（Domain Experts）极难参与数据的共创与更新。

**愿景目标**：
打造一款**前端极致轻量、基于无限画板拓扑、绝对数据安全**的可视化“本体协作中台”。它需要彻底摆脱枯燥的表格或表单，以**极致的拖拉拽图谱体验 (Visual Node-Graph Editor)**，让业务专家进行画布级别的结构延展，并最终自动转化为“GitHub库开源代码提交”的自动化管道。

---

## 2. 用户角色 (User Personas)

1. **业务领域专家 (Domain Expert / Business Analyst)**
   * **特征**：不懂代码，完全靠空间感和业务蓝图来设计架构。
   * **诉求**：像使用 Miro 或 Figma 一样在极简画板上拉出点与线，将自己的业务场景映射出来。
2. **数据架构师 (Data Architect / Repository Maintainer)**
   * **特征**：负责梳理系统接入与审核代码仓库。
   * **诉求**：不仅要在 GitHub 里轻松点合并确认，更重要的是能在 L0 平台通过拉线完成物理系统到业务本体的数据挂载（数据血缘）。

---

## 3. 核心功能与模块规划 (Functional Requirements)

### 3.1 身份鉴权与同步 (Authentication & GitOps)
| 功能 ID | 功能名称 | 描述 |
| :--- | :--- | :--- |
| AUTH-01 | GitHub 鉴权接入 | 采用 GitHub OAuth 拦截入场，读取鉴权用户的提交权限。 |
| GIT-01 | 画布到 JSON 的转换器 | 当在画布点击【提交】，前端图谱的 Node/Edge 坐标与属性会被实时反拍回严格的 `universal_ontology_v1.json` 规则树中，并代用户建立 Pull Request (PR)。 |

### 3.2 L1/L2/L3：多维宇宙拖拽网格编辑器 (Visual Drag-and-Drop Ontology Editor)
*传统表单输入被摒弃，全面采用 React Flow 或 G6 等流式拓扑画布驱动。*
| 功能 ID | 功能名称 | 描述与规则 |
| :--- | :--- | :--- |
| ED-01 | L1 通用核心画板 | 架构师作为主要操作者。在无尽画布上拖拉出所有跨行业的核心原子节点（Entity, Party 等），以及用连线定义核心泛化关系。 |
| ED-02 | L2 / L3 继承映射工作台 | 分析师在进入 L2/L3 时，画布会“高亮锁定”上一层的结构作为只读起点约束，用户只能**从继承节点（Parent Node）上拖拽连线生成出自己层级独有的特化节点**。保障下不犯上，结构清晰。 |
| ED-03 | 悬浮动态属性侧边栏 | 当在画板选区单击一个“概念节点”或“连线”时，右侧划出极简黑色侧栏面板，快速填写该对象的定义语、翻译字段、及具体参数（Data Types）。 |

### 3.3 L0：系统物理挂载探针 (L0 System Data Wire & Mapping)
*L0 不再是纯本体图，而是承载着逻辑业务域与 IT 系统域映射边界的接线盒 (Junction Box)。*
| 功能 ID | 功能名称 | 描述与规则 |
| :--- | :--- | :--- |
| L0-01 | 跨系统集成接线索 | L0 画布被分为两半。左侧显示系统外部对接获取到的数据库表/字段接口，右侧显示 L1~L3 定义好的规范化概念。架构师可以像插拔跳线一样**将实体数据源用拖线的方式连向业务本体**，生成标准的解析器映射表。 |
| L0-02 | 数据源端点接入器 | 支持解析并读取一些外部数据库名和字段，形成供人推拉的实体源锚点。 | 

---

## 4. 技术栈选型补丁 (Tech Stack Upgrade)

基于上述全新推翻的可视化极客操作流需求：
* 我们将在前端额外引入底层图形库 **React Flow**。
* 设计上将采用节点微动画交互连线（比如引出关系时有电流动效或绿色高亮约束吸附感），极大凸显您要求的“德勤科技极简风”的空间张力。
